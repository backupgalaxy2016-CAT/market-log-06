## GitHub Pages에 올리는 방법

1. GitHub에서 새 저장소를 만듭니다 (예: `market-log`). **Public**으로 생성합니다.
2. 이 폴더의 `index.html`과 `README.md`를 저장소에 업로드합니다.
   - 웹에서: 저장소 → `Add file` → `Upload files` → 파일을 끌어다 놓고 `Commit changes`
3. 저장소 → `Settings` → 왼쪽 메뉴 `Pages` → **Branch: main / (root)** 선택 → `Save`
4. 1~2분 뒤 `https://내아이디.github.io/market-log/` 주소로 접속하면 완성입니다.

## 사용 흐름 (매일)

1. **오늘 기록** 탭에서 ①지수 → ②주도주 → ③비주도주 → ④아시아 → ⑤미국 순서로 입력
2. 차트 스크린샷을 첨부 (KOSPI, 주도주, 미국 지수 캡처)
3. **자동 판정 실행**으로 오늘의 판정과 위험도 게이지 확인
4. **일지 저장** → 일지 목록에서 날짜별로 다시 볼 수 있음

## 데이터 보관에 대해

- 기록은 **브라우저(localStorage)**에 저장됩니다. 같은 브라우저에서는 계속 유지되지만, 브라우저를 바꾸거나 데이터를 지우면 사라집니다.
- 사이드바의 **백업(JSON)**으로 주기적으로 내려받아 두세요. 다른 기기에서는 **복원**으로 불러올 수 있습니다.
- 일지 상세 화면의 **MD 다운로드**로 마크다운 파일을 받아, 저장소에 `logs/2026-07-05.md`처럼 커밋하면 GitHub 자체에도 일기처럼 기록이 쌓입니다.

## 시세 자동 연동에 대해

GitHub Pages는 정적 페이지라서 KOSPI/KOSDAQ 실시간 시세를 무료로 안정적으로 가져올 방법이 없습니다(거래소 API는 키·서버 필요, 무료 API는 CORS 차단). 대신:

- 종가와 등락률만 직접 입력하거나 (30초면 충분),
- HTS/네이버 차트 **스크린샷을 Claude에 올리면** 수치를 읽고 판정 초안까지 만들어 줍니다. 그 결과를 이 페이지에 옮겨 저장하면 됩니다.
